Simulate the current S\+B\+ML model. 

If options is null, then the current simulation settings (start time, end time, n steps) are used. If options is not null, then the current simulation settings are set to the values specified by options and they are used.

The options \mbox{\hyperlink{classrr_1_1Dictionary}{Dictionary}} may contain a large number of options, for a complete list of all available options for each integrator type, \begin{DoxySeeAlso}{See also}
\mbox{\hyperlink{classrr_1_1IntegratorFactory}{Integrator\+Factory}}, 

\mbox{\hyperlink{classrr_1_1SimulateOptions}{Simulate\+Options}}.
\end{DoxySeeAlso}
For example, to perform a simulation from time 0 to 10 with 1000 steps, using a stiff integtator, you would\+: 
\begin{DoxyCode}{0}

\DoxyCodeLine{opt.steps = 1000;}

\end{DoxyCode}


Similarly, options specific to a particular integrator, such as the \textquotesingle{}seed\textquotesingle{} option with the Gillespie integrator, this is set via the \textquotesingle{}set\+Integrator\textquotesingle{} \char`\"{}integrator\char`\"{} key, i.\+e. 
\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{classrr_1_1RoadRunner_a04a6fd481cfacc2510d5d113520143ef}{RoadRunner}} rr = \mbox{\hyperlink{classrr_1_1RoadRunner_a04a6fd481cfacc2510d5d113520143ef}{RoadRunner}}(\textcolor{stringliteral}{"someFile.xml"});}

\DoxyCodeLine{SimulateOptions opt;}
\DoxyCodeLine{opt.start = 0;}
\DoxyCodeLine{opt.duration = 10;}
\DoxyCodeLine{opt.steps = 1000;}
\DoxyCodeLine{opt.setItem(\textcolor{stringliteral}{"stiff"}, \textcolor{keyword}{true});}
\DoxyCodeLine{opt.setItem(\textcolor{stringliteral}{"seed"}, 12345);}
\DoxyCodeLine{\textcolor{keyword}{const} DoubleMatrix *result = rr.simulate(\&opt);}

 Here, the \char`\"{}integrator\char`\"{} specifies the integrator to use. The \char`\"{}stiff\char`\"{} key is only used by the deterministic solvers, and it is safely ignored by the stochastic solvers. Also, the \char`\"{}seed\char`\"{} sets the random seed that the integrator uses. For more information about all of the available options for each integrator, \begin{DoxySeeAlso}{See also}
Integrator\+Factory\+::get\+Integrator\+Options".
\end{DoxySeeAlso}
If one wants to not store the result matrix in memory and instead write it to a file during simulation, one can set the output\+\_\+file option. When output file is nonempty, \mbox{\hyperlink{classrr_1_1RoadRunner_a377834f22edadb7706779ecca7c1eb4e}{simulate()}} will write to its path once every Config\+::\+K\+\_\+\+R\+O\+W\+S\+\_\+\+P\+E\+R\+\_\+\+W\+R\+I\+TE rows are generated and clear the matrix. So an empty result matrix is returned, and the last simulation results are not stored.
