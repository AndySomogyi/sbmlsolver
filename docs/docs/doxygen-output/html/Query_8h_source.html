192  /*
193  * @brief returns query results in a map where keys are
194  * the variable names used in query and values are vectors
195  * of values for results.
196  *
197  * @details This is the only supported mechanism for ascertaining
198  * query results programatically, without needing to read/write to/from
199  * file. This method uses an iterator from librdf. When it gets to the end
200  * of the iterator, the iterator is "used up". Therefore at the end of
201  * collecting the results as a map, the original query is re-run, so that
202  * the presently instantiated Query object is not rendered useless.
203  *
204  * If the query looks like this:
205  * @code
206  * std::string query_string = "SELECT ?x ?y ?z
207  * "WHERE {"
208  * " ?x ?y ?z"
209  * "}";
210  * Query query(query_string);
211  * @endcode
212  * The column names will be x, y and z and the results
213  * will be string representations of all subjects (?x),
214  * predicates (?y) and resources (?z) in a vector of size `n`,
215  * where n is the number of query results.
216  */
217  ResultsMap resultsAsMap();
218
